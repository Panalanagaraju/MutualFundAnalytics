# Understanding the Data

The Fund Master dataset acts as the foundation of the project because it contains scheme-level information such as AMFI codes, fund houses, categories, expense ratios, risk categories, and fund managers.

The NAV History dataset contains daily NAV values for all schemes and will be used later for return calculations, performance analysis, and risk measurement.

The AUM dataset provides information about the assets managed by different fund houses and will help in market share analysis.

The SIP dataset contains monthly SIP inflows and active SIP account information, which will be useful for understanding investment trends and investor participation.

The Investor Transactions dataset is particularly interesting because it contains demographic details such as state, city, age group, gender, and income level. This dataset will support the investor behaviour and demographic analysis planned for later stages of the project.

The Portfolio Holdings dataset provides stock-level holdings information that can be used to analyze sector allocation and portfolio concentration.

Finally, the Benchmark dataset contains historical index values that will later be used to calculate performance metrics such as Alpha, Beta, Sharpe Ratio, and tracking error.

# Data Quality Validation

One of the major objectives of Day 1 was to ensure that the datasets were clean and reliable.

I performed several validation checks, including:

* Missing value analysis
* Duplicate record detection
* Data type verification
* File integrity checks

The results were encouraging. No duplicate records were found in any dataset, and all files loaded successfully without errors.

The only issue identified was in the Monthly SIP Inflows dataset, where twelve values were missing in the Year-over-Year Growth column. After reviewing the data, this appears to be expected because YoY growth cannot be calculated during the initial months when previous-year values are unavailable.

Therefore, these missing values are considered valid and do not affect the quality of the dataset.

Overall, the datasets were found to be highly clean and well-structured.

# Understanding AMFI Scheme Codes

During the analysis, I also examined the AMFI scheme codes available in the Fund Master dataset.

I observed that every mutual fund scheme is assigned a unique AMFI code, which serves as a common identifier across datasets and API responses. These codes will be extremely important during database design because they will act as the primary linking field between different tables.

This understanding will help significantly when building the star schema and ETL pipeline in the next phase of the project.
