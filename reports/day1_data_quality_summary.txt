Day 1 Data Quality Summary

Today, I completed the initial data ingestion task for the Mutual Fund Analytics project. After setting up the project structure and installing the required Python libraries, I fetched mutual fund NAV data from the MFAPI service and stored the data as CSV files in the data/raw folder.

The following CSV files were loaded and analyzed:

* Axis_Bluechip.csv
* HDFC_Top100.csv
* ICICI_Bluechip.csv
* Kotak_Bluechip.csv
* Nippon_LargeCap.csv
* SBI_Bluechip.csv

I used Pandas to load each file and checked the shape, data types, sample records, missing values, and duplicate records.

The number of records in each dataset is shown below:

* Axis_Bluechip.csv – 3,579 rows and 4 columns
* HDFC_Top100.csv – 3,105 rows and 4 columns
* ICICI_Bluechip.csv – 3,321 rows and 4 columns
* Kotak_Bluechip.csv – 3,315 rows and 4 columns
* Nippon_LargeCap.csv – 3,312 rows and 4 columns
* SBI_Bluechip.csv – 3,250 rows and 4 columns

The datasets contain four common fields: date, nav, scheme_code, and scheme_name. The NAV values are stored as floating-point numbers, while scheme codes are stored as integers. The date column is currently stored as text and will need to be converted to datetime format before carrying out time-series analysis.

A missing value check was performed on all datasets. No missing values were found in any of the columns, which indicates that the downloaded data is complete and suitable for further processing.

I also checked for duplicate records in every CSV file. No duplicate rows were detected. This confirms that the data has been stored correctly and does not contain repeated entries.

While reviewing the sample records, I noticed that some scheme names returned by the API did not exactly match the names expected from the file names. For example, HDFC_Top100.csv contains records for "SBI Small Cap Fund - Direct Plan - Growth", and SBI_Bluechip.csv contains records related to another mutual fund scheme. This observation suggests that the scheme codes should be validated against official AMFI records before proceeding with detailed analysis.

Overall, the data ingestion process was completed successfully. All six CSV files were loaded without errors, no missing values were found, and no duplicate records were detected. The datasets are now ready for preprocessing, validation, and exploratory analysis in the next phase of the project.
